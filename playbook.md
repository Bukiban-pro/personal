# Java Fresher / Intern Extreme Playbook (Final Version)

_Last updated: March 2026_

This is a **practical, step‑by‑step system** to prepare for Java fresher / intern backend interviews.

- No vague suggestions.
- Fixed links and fixed daily tasks.
- Uses only verified free resources (as of March 2026).

---

## 0. Rules

1. **Prepare for patterns, not leaked questions**
   - Companies change specific questions often.
   - They consistently test: aptitude/logic, arrays/strings, basic trees, Java/OOP, Spring, SQL, and behavioral stories.

2. **Two modes every day**
   - **Passive Mode (low energy)**
     - Watch 1 specific video from this playbook.
     - Write exactly 3 bullet notes.
   - **Active Mode (normal energy)**
     - Do 1–2 specific tasks (problems/tests/notes) listed for that day.
   - On bad days, Passive only is acceptable. The rule is: do something.

3. **Realistic timeline**
   - If you already know some Java and some DSA: expect **4–6 weeks** to feel solid.
   - If you are rusty or new to DSA: expect **8–10 weeks**.

4. **Minimal tracking**
   - File: `Notes/progress.md`.
   - Each day, append one line:
     - `YYYY-MM-DD – Passive: [what] – Active: [what] – Mood: [1 word]`.
   - The important metric is days with any work.

## 4. Daily Operating System

Each day:

1. Open `progress.md` and start a new line with today’s date.
2. Decide: **Passive only** or **Passive + Active**.
3. Follow the script for that calendar day (below).
4. Complete the line in `progress.md` with what you did.

If energy is very low, do the Passive part only. That still counts.

---

## 5. 4‑Week Core Plan

You can repeat, compress, or stretch this cycle depending on your timeframe.

### Week 1 – Aptitude and Code Reading

**Day 1 – Numerical basics**
- Passive:
  - Watch the main numerical reasoning tutorial (S3ALlTH9ohc).
- Active:
  - Register/log in to Graduates First.
  - Take 1 free numerical reasoning test.
  - In `aptitude.md`, write:
    - 3 types of questions you saw.
    - 3 mistakes you made.

**Day 2 – More numericals / practice**
- Passive:
  - Watch the second numerical reasoning video or 1–2 videos from the numerical reasoning playlist.
- Active:
  - Take another numerical or mixed aptitude test, or re‑attempt similar questions.
  - Update `aptitude.md` with the main patterns you see.

**Day 3 – Logical reasoning**
- Passive:
  - Watch one logical reasoning video from the same or similar aptitude channel.
- Active:
  - Take 1 logical or mixed aptitude test.
  - Note 3 logic patterns in `aptitude.md` (e.g. sequences, grouping, true/false).

**Day 4–5 – Code reading (arrays/strings)**
- Passive:
  - Watch the DSA patterns video up to arrays/strings/hashmaps, or rewatch the relevant section.
- Active:
  - On LeetCode, open:
    - Two Sum
    - Contains Duplicate
    - Valid Anagram
  - For each problem:
    - Read one clear solution.
    - In `code_reading.md`, write:
      - Problem name.
      - 2–3 sentences: what the algorithm does.
      - Time and space complexity.

**Day 6–7 – Arrays/Strings practice**
- Passive:
  - Watch the arrays & strings crash course video (0OQJDd3QqQM).
- Active:
  - Solve 1–2 easy array/string problems per day (NeetCode easy or similar LeetCode problems).
  - Add each solved problem to `DSA_core.md` with columns: Problem | Idea | Time | Space.

### Week 2 – DSA Fundamentals (Trees + More Arrays)

**Goal:** have at least 8–12 easy problems you can explain clearly.

**Day 8–10 – More arrays/strings**
- Passive:
  - Rewatch relevant parts of the DSA patterns video.
- Active:
  - Choose 2–3 more array/string problems from NeetCode’s easy list.
  - For each:
    - Try 15–20 minutes.
    - Read a solution.
    - Reimplement from memory.
    - Record in `DSA_core.md`.

**Day 11–14 – Trees & stacks**
- Passive:
  - If needed, watch a short explanation for invert binary tree or tree recursion.
- Active:
  - Solve, across these days:
    - Maximum Depth of Binary Tree
    - Invert Binary Tree
    - Valid Parentheses
  - For tree problems, sketch a tree and trace recursion by hand.

### Week 3 – Java OOP, Collections, Exceptions

**Day 15 – Java OOP overview**
- Passive:
  - Watch the Java OOP interview video (yL1D0opZwwA).
- Active:
  - Open InterviewBit Java questions.
  - In `Java_OOP.md`, write your own 1‑sentence definitions for:
    - Class, object, encapsulation, inheritance, polymorphism, abstraction.

**Day 16–17 – OOP practice**
- Passive:
  - Rewatch unclear sections of the OOP video.
- Active:
  - From InterviewBit, pick questions on:
    - Interface vs abstract class.
    - Overloading vs overriding.
  - For each concept, write a small Java snippet in `Java_OOP.md`.

**Day 18–19 – Collections**
- Passive:
  - Watch a concise Java collections overview video.
- Active:
  - From InterviewBit or Java docs, write in `Java_OOP.md`:
    - When to use List vs Set vs Map.
    - Differences between ArrayList and LinkedList.
    - Basic HashMap behavior.
  - Turn these into about 10 Q&A flashcards.

**Day 20–21 – Exceptions**
- Passive:
  - Watch a short video on checked vs unchecked exceptions.
- Active:
  - From InterviewBit, summarize:
    - Checked vs unchecked exceptions, with 1 example each.
    - Basics of try–catch–finally and throws.
  - Write 2–3 small code snippets showing exception handling in `Java_OOP.md`.

### Week 4 – Spring, SQL, Behavioral

**Day 22 – Spring Boot intro**
- Passive:
  - Watch the 5‑minute Spring Boot REST API video.
- Active:
  - In `Spring_SQL.md`, write:
    - What Spring Boot is.
    - What `@RestController` does.
    - What dependency injection means.

**Day 23–24 – Simple REST API**
- Passive:
  - Watch the 10‑minute CRUD Spring Boot REST API video.
- Active:
  - Create a Spring Boot project via Spring Initializr.
  - Implement a simple entity (e.g. `Book`) and two endpoints:
    - `GET /books`
    - `POST /books`
  - Note what each annotation does in `Spring_SQL.md`.

**Day 25 – SQL joins**
- Passive:
  - Watch the SQL Joins tutorial (2HVMiPPuPIM).
- Active:
  - In `Spring_SQL.md`, write:
    - Two INNER JOIN examples.
    - One LEFT JOIN example.

**Day 26–27 – Behavioral / STAR**
- Passive:
  - Watch the STAR method video (dRqN4BuhCHU).
- Active:
  - In `projects.md`, write two STAR stories from your own projects.
  - Say each story out loud at least twice.

**Day 28 – Review and plan**
- Passive:
  - Rewatch the weakest part from any required video.
- Active:
  - In `progress.md`, note:
    - Topics you feel strong in.
    - Topics that still feel weak.
  - Turn the weak list into a short plan for the next cycle.

## 6. Pre‑Interview 90‑Minute Routine

Use this script before any important test or interview.

1. **25 min – Aptitude**
   - Do a short numerical/logic test or a block of practice questions.
2. **25 min – DSA & Java**
   - Skim `DSA_core.md`; re‑explain 5 problems in your head.
   - Skim `Java_OOP.md`; restate OOP pillars and basic collections.
3. **20 min – Spring & SQL**
   - Skim `Spring_SQL.md`; recall what main annotations do.
   - Re‑explain your JOIN examples from memory.
4. **20 min – Behavioral**
   - Read 2 STAR stories from `projects.md`.
   - Say them out loud once.

---

## 7. After Each Test / Interview

Immediately after finishing:

- Open `interview_log.md` and record:
  - Company, role, date.
  - Rounds you had (aptitude, coding, Java/Spring, behavioral).
  - 3 things you were not confident about.
  - 1 thing you did well.

Over the next few study days:

- Add at least one Active task aimed at those weak points.
- Example: if trees were hard, add one tree problem per relevant day.

---

## 8. Adjusting for Time

- **If you only have ~2 weeks**:
  - Focus on:
    - Week 1 (aptitude + arrays/strings + DSA patterns overview).
    - Week 3, days 15–17 (core Java OOP).
    - Week 4 key days (basic Spring and SQL joins).
- **If you have 2+ months**:
  - Repeat the 4‑week cycle.
  - Increase the number of DSA problems, build more small Spring projects, and add mock interviews.

The main objective is consistency. If you maintain the daily habit, this playbook covers the core skills companies actually test at fresher Java backend level.
